<!--
This is one of the classic problems with npm's dependency resolution.

When two packages request different major versions of the same dependency, npm can't hoist one copy to the root. Instead, it nests copies inside each package's node_modules.

This means:
- More disk space used
- More packages to audit for CVEs
- More copies of the same code with the same vulnerabilities
- Each copy needs a separate patch

Tools like pnpm solve this with symlinks, but the duplication problem persists across the ecosystem.
